JSON-RPC Proxy

POST /

Standard Ethereum JSON-RPC endpoint. Proxies all methods to a local Geth node, except eth_sendRawTransaction which is routed through Flashbots Protect for sandwich attack prevention.

Base URL

https://rpc.reclaimfi.xyz

Request

Standard JSON-RPC 2.0 format. Single requests and batch arrays are both supported.

// Example: get latest block number { "jsonrpc": "2.0", "method": "eth_blockNumber", "params": [], "id": 1 }

Response

{ "jsonrpc": "2.0", "result": "0x134a7b2", "id": 1 }

GET /health

GET /health

Health check endpoint. Verifies the proxy is running and the Geth backend is reachable and synced. Returns HTTP 200 when healthy, HTTP 503 when degraded.

Response Schema

{ "status": "healthy", // "healthy" or "degraded" "geth_connected": true, // Whether Geth backend responded "geth_block": 20148291, // Latest block number from Geth "uptime_seconds": 864000, // Proxy uptime "version": "0.1.0" // Proxy version }

Frequently Asked Questions

What is MEV?
MEV (Maximal Extractable Value) is the profit that block builders and searchers extract from ordinary users by reordering, inserting, or censoring transactions within a block. The most common form is the sandwich attack: a bot sees your DEX swap in the public mempool, places a buy order before yours (front-run) to push the price up, lets your trade execute at the worse price, then sells after (back-run) to pocket the difference. On Ethereum, MEV extraction costs users an estimated $500M+ per year.
How does Reclaim protect my transactions?
When you send a transaction through Reclaim, we route it via Flashbots Protect instead of the public mempool. Your transaction is sent directly to block builders through a private channel, so sandwich bots never see it. This is the same infrastructure used by professional traders and MEV searchers. If Flashbots Protect is temporarily unreachable, we automatically fall back to direct Geth submission so your transaction always goes through.
How do rebates work?
When your swap creates a backrun opportunity (a profitable arbitrage that can be placed immediately after your trade), Reclaim captures that value and returns 80% to you as a rebate. This is MEV that would have been captured by searchers anyway, but instead of going to anonymous bots, it comes back to you. Rebates are distributed on-chain via our smart contract. You don't need to claim them -- they are sent to your wallet automatically.
Is it free?
Yes, completely free, forever. There is no subscription, no per-transaction fee, and no hidden costs. Our revenue model is simple: we keep 20% of the MEV we capture from backruns. You get 80%. If we don't capture any MEV, nobody pays anything. Your transactions are always protected regardless of whether MEV is captured.
Is it open source?
Yes. The RPC proxy, website, and smart contracts are all open source. You can audit every line of code that handles your transactions.

What wallets are supported?
Any Ethereum wallet that allows you to set a custom RPC endpoint. This includes:

Rabby -- Settings > Custom RPC

Rainbow -- Settings > Networks > Custom RPC

Taho -- Networks > Add custom RPC

Use RPC URL https://rpc.reclaimfi.xyz with Chain ID 1.
Is my data collected?
No. Reclaim does not track users, store personal data, or collect analytics. The only data we see are the Ethereum transactions you submit through our RPC -- the same data any RPC provider sees. We do not log IP addresses, we do not use cookies, and we do not share any data with third parties. Transaction data is processed in memory and not persisted beyond operational stats counters.
What if Reclaim goes down?
The proxy has a built-in fallback mechanism. If Flashbots Protect is unreachable, your transaction is automatically submitted directly to our local Geth node. If the entire proxy is offline, you can temporarily switch back to any public Ethereum RPC (Infura, Alchemy, etc.) in your wallet settings. No transactions are ever lost -- they either go through Protect, fall back to Geth, or wait in your wallet for you to resubmit.
How do referrals work?
Connect your wallet on reclaimfi.xyz to get a referral link. When someone uses Reclaim through your link:

They get 85% of captured MEV (5% more than the standard 80%)
You get 5% of their captured MEV as passive income
Reclaim keeps 10% (vs the usual 20%)

Referral payouts are handled on-chain via the Reclaim smart contract. No caps, no expiry. Earnings accumulate forever for every referred user.

Reclaim runs on dedicated bare-metal infrastructure with a full Ethereum execution + consensus client stack. No third-party RPC dependencies.

Infrastructure

  • Hardware: Ryzen 9 5950X (16c/32t), 128GB ECC RAM, NVMe storage
  • Execution client: Geth (full sync, 5TB NVMe)
  • Consensus client: Lighthouse
  • Proxy: FastAPI + Uvicorn (4 workers)
  • TLS: Nginx reverse proxy with Let's Encrypt
  • Protection: Flashbots Protect (fast mode)

Rebate Distribution

  • Trigger: Rebates are calculated whenever a backrun bundle is successfully included in a block.
  • Minimum threshold: 0.001 ETH profit required before a backrun is attempted.
  • Distribution: On-chain via the Reclaim smart contract. Sent directly to the user's wallet.
  • Timing: Typically within 1-2 blocks of the original swap.
  • Tracking: All rebate transactions are visible on Etherscan under the contract address.
