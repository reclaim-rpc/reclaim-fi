# Reclaim Fi -- n8n Marketing Workflow Orchestrator

n8n is the workflow automation engine powering Reclaim's automated marketing, community management, and analytics pipelines.

## Quick Start

```bash
# Automated setup (generates password, starts n8n)
bash setup.sh

# Or manually:
docker compose up -d
```

## Access

- **URL:** http://135.181.61.221:5678
- **Username:** `reclaim-admin`
- **Password:** See `.env` file (auto-generated by `setup.sh`)

## Commands

```bash
# Start
docker compose up -d

# Stop
docker compose down

# View logs
docker compose logs -f

# Restart
docker compose restart

# Update to latest n8n
docker compose pull && docker compose up -d
```

## Data Persistence

All n8n data (workflows, credentials, execution history) is stored in:
```
/root/reclaim-fi/n8n/data/
```

This directory is mounted as a Docker volume. Back it up before major updates.

## Workflows to Create in n8n UI

After setup, create these five workflows via the n8n web interface:

### 1. Daily Content Pipeline

**Trigger:** Cron -- every 6 hours (0 */6 * * *)

**Flow:**
1. HTTP Request node: `GET https://rpc.reclaimfi.xyz/stats` -- fetch live metrics
2. HTTP Request node: `POST` to marketing API (or direct Anthropic API call) -- generate content based on stats
3. Conditional branches:
   - Twitter: Post via Twitter API (OAuth 1.0a credentials)
   - Reddit: Post/comment via Reddit API
   - Telegram: Send to channel via Bot API
   - Discord: Post via webhook

**Key data points to include in prompts:**
- `total_txs_protected` -- social proof
- `total_mev_captured_eth` -- value captured
- `total_rebates_paid_eth` -- rebates distributed
- `active_users` -- growth metric

### 2. Community Response Bot

**Trigger:** Webhook (POST to n8n webhook URL)

**Flow:**
1. Receive incoming message/mention via webhook
2. Classify intent (use AI node or simple keyword matching):
   - Support question -> Generate helpful response
   - Complaint -> Escalate + generate empathetic response
   - Feature request -> Log + acknowledge
   - Spam -> Ignore
3. Route response to appropriate platform (Twitter reply, Reddit comment, etc.)

**Setup:** Configure platform monitoring to POST mentions to the webhook URL.

### 3. Growth Analytics

**Trigger:** Cron -- daily at 00:00 UTC

**Flow:**
1. HTTP Request: `GET https://rpc.reclaimfi.xyz/stats` -- current metrics
2. Read previous day's snapshot from n8n storage or file
3. Calculate deltas: new users, tx growth, MEV captured growth
4. Generate markdown report with:
   - Daily active users
   - Transaction volume (total + protected)
   - MEV captured and rebates paid
   - Top RPC methods by volume
   - Week-over-week and month-over-month trends
5. Send report to Telegram/Discord channel
6. Save current snapshot for next comparison

### 4. Partnership Outreach

**Trigger:** Cron -- weekly (Monday 10:00 UTC)

**Flow:**
1. Read outreach target list (wallet providers, DEX aggregators, dashboards, DAOs)
2. Check outreach log to avoid duplicate contacts
3. For each new target:
   - Generate personalized outreach message using AI node
   - Include latest Reclaim stats as proof points
   - Tailor pitch angle to target category
4. Queue messages for manual review (or auto-send if configured)
5. Log outreach attempt with timestamp

**Target categories:**
- Wallet providers (MetaMask, Rabby, Frame, Rainbow, Taho)
- DEX aggregators (1inch, Paraswap, CowSwap, Matcha, KyberSwap)
- DeFi dashboards (Zapper, DeBank, Zerion)
- Content creators (Finematics, Bankless, The Defiant)
- DAO treasuries (Aave, Uniswap, ENS, Lido)

### 5. Competitor Monitoring

**Trigger:** Cron -- every 12 hours

**Flow:**
1. For each competitor RPC endpoint:
   - Flashbots Protect: `https://rpc.flashbots.net`
   - MEV Blocker: `https://rpc.mevblocker.io`
   - SecureRPC: `https://api.securerpc.com/v1`
2. Send `eth_blockNumber` JSON-RPC call, measure response time
3. Check if their stats endpoints expose useful data
4. Compare latency vs Reclaim RPC
5. Generate comparison report:
   - Response times
   - Uptime status
   - Feature differences (rebates, referrals, etc.)
6. Alert if a competitor launches a new feature or changes pricing
7. Send report to internal channel

## Environment Variables

| Variable | Description |
|----------|-------------|
| `N8N_PASSWORD` | Basic auth password (auto-generated) |
| `GENERIC_TIMEZONE` | Timezone for cron triggers (UTC) |
| `N8N_METRICS` | Enable Prometheus metrics endpoint |
| `WEBHOOK_URL` | External URL for webhook triggers |

## Security Notes

- Basic auth protects the n8n UI -- do not expose port 5678 without it
- Store API keys (Twitter, Anthropic, etc.) in n8n's built-in credential store, not in environment variables
- The `.env` file has `chmod 600` -- keep it restricted
- Consider adding a reverse proxy with TLS for production use
